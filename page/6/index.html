字数统计 | 阅读时长

手机助手的清理模块有一个功能需要获取存储空间使用比例.

网上找的方法大概是,使用时传入sd卡根目录:

    public static long[] getTotalAndAvailableBytes(String path) {
    if (TextUtils.isEmpty(path)) {
        return null;
    }
    try {
        File pathFile = new File(path);
        StatFs stat = new StatFs(pathFile.getAbsolutePath());
        if (Build.VERSION.SDK_INT >= 18) {
            long total = stat.getBlockCountLong() * stat.getBlockSizeLong();    //磁盘总大小,单位byte
            long available = stat.getAvailableBlocksLong() * stat.getBlockSizeLong();   //磁盘剩余空间,单位byte
            return new long[] {total, available};
        }
        long total = stat.getBlockCount() * stat.getBlockSize();
        long available = stat.getAvailableBlocks() * stat.getBlockSize();
        return new long[] {total, available};
    } catch (Exception e) {
        return null;
    }
}

粗看,已经是做了版本适配的方法,挺靠谱的.

hexo的_config.yml文件中一项配置值是网站描述:

# 网站标题
title: 100斤的博客
# 网站副标题
subtitle: android,linux
# 网站描述
description: 请在场的各位给个面子.<br/>——香克斯  
author: 100斤

但是在next主题Pisces情况下完全不显示.搜了一会没找到方法,只能自己动手了.

方法记录如下:

在theme/layout/_macro/sidebar.swig中 if theme.links之前添加如下内容(去掉html标签中的#号),

{% if config.description %}
        <#div class="custom-logo-site-title">
            <#p class="site-state-item-count">{{ config.description }}<斜杠p>
        <斜杠div>
    {% endif %}

初始数据库脏读

发表于 2017-08-21 | 更新于 2017-12-15 | 分类于 技术杂谈 |
字数统计 | 阅读时长

平时数据库用的少,掌握的知识仅限理论,一直想提高一下.项目上线后稍微轻松了一点,抓住了后台的兄弟学习(聊天).第一次见识到了后台的数据库脏读.

后台有一个接口,是一系列开关的配置值,每次更新的时候会先删除数据库中的配置,然后插入新的配置.在删除配置时,如果缓存失效,从数据库拿,就会取到空作为缓存,然后问题就来了.缓存失一个失效时间,在这个时间内,更新的配置值都不会生效.

虽然原理很简单,一旦没处理好,这也是后台的一个事故.
