英语

发表于 2017-09-01 | 更新于 2017-12-15 | 分类于 吐槽 |
字数统计 | 阅读时长

英语对于程序员的重要性我从来没有怀疑过.

但是在有中文的情况下,为了快速学习,有时还是会选择中文的文档.在翻swig官网的时候就踩了这个坑.对比一下中文首页,可以看到news相差非常大,英文的news是最新的,而中文的只更新到2010年,让我一时误以为这个东西可能都已经过时了!!!

不过对于这种开源项目也不能过分苛责,毕竟多语言这个东西可是很贵的!免费维护一个项目已经是不易,再花经费就更加肉疼了.所以还是培养一下自身的英语阅读能力好了.

对于写代码的人,英语不止阅读,还有写作.

类,方法命名的时候,有不错的英语能力对代码的可读性提高是很好的.想比于设计模式,在方法命名的时候多琢磨一下,都使项目的维护更加简单.

整型溢出

发表于 2017-08-30 | 更新于 2017-12-15 | 分类于 技术杂谈 |
字数统计 | 阅读时长

手机助手的清理模块有一个功能需要获取存储空间使用比例.

网上找的方法大概是,使用时传入sd卡根目录:

    public static long[] getTotalAndAvailableBytes(String path) {
    if (TextUtils.isEmpty(path)) {
        return null;
    }
    try {
        File pathFile = new File(path);
        StatFs stat = new StatFs(pathFile.getAbsolutePath());
        if (Build.VERSION.SDK_INT >= 18) {
            long total = stat.getBlockCountLong() * stat.getBlockSizeLong();    //磁盘总大小,单位byte
            long available = stat.getAvailableBlocksLong() * stat.getBlockSizeLong();   //磁盘剩余空间,单位byte
            return new long[] {total, available};
        }
        long total = stat.getBlockCount() * stat.getBlockSize();
        long available = stat.getAvailableBlocks() * stat.getBlockSize();
        return new long[] {total, available};
    } catch (Exception e) {
        return null;
    }
}

粗看,已经是做了版本适配的方法,挺靠谱的.
