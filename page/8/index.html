ubuntu拨号上网

发表于 2017-08-19 | 更新于 2017-12-15 | 分类于 Linux |
字数统计 | 阅读时长

linux这个东西,怎么说的呢,功能很强,但是用户体验是真的垃圾.

刚搬的房子开了个网,还没搞路由器,想先用电脑试一下,需要拨号上网.打开了网络设置,添加网络,出来个列表

- VPN
- 绑定(Bond方式)
- 绑定(Team方式)
- 桥接
- VLAN

懵逼了,DSL呢????只能谷歌了,然后一堆命令开启拨号连接,记录如下:

# 初始化
sudo apt-get install pppoeconf
sudo pppoeconf

# 联网
sudo pon dsl-provider

# 断网
sudo poff

# 查看日志
plog

# 查看接口信息
ifconfig ppp0

所以linux这个东西真的不建议小白使用,不会用的话办公效率还不如windows,以上.

android studio的坑二

发表于 2017-08-16 | 更新于 2018-08-19 | 分类于 Android |
字数统计 | 阅读时长

android studio如果有过期的API会提示depressed,有新的api会提示new api,这种设计很好的帮助了开发者.

然而这种良好的设计是有bug的.尤其是在布局文件中.

目前发现了两个:

1.singleline这个属性会被提示depressed,使用maxline="1"替代,
然而低版本maxline这个属性其实没有生效.

2.text的gravity在低版本不能适配左右布局,也没有什么提示.
低版本需要配合textAlignt使用.

记android studio的一个坑

发表于 2017-08-11 | 更新于 2018-08-19 | 分类于 Android |
字数统计 | 阅读时长

android开发的时候如果需要开发相似的界面,可能会直接copy一份然后改一改,调一调.

今天就遇到了上面的情况,copy了一份,改了一下view的id,但是漏了几个.

但是编译器没报错,代码运行也没crash,就是view的位置不对.

原因是这样的:layout中的id,可以指向不是当前view的id,编译器不会检查,运行也会被忽略.

例如relativelayout中layout_below的id可以随便指一个,只要在项目中….好像这个坑还是蛮严重的.敲代码头晕眼花就怎么都发现不了.难受.
