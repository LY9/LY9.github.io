android:largeHeap缓解OOM实践

发表于 2017-10-19 | 更新于 2017-12-15 | 分类于 Android |
字数统计 | 阅读时长

在Android开发中，我们必然遇到过OutOfMemoryError这样的崩溃，产生的原因无外乎两点，一是内存过小不够用，二是程序设计有误，导致不能及时释放内存。在第一种情况的时候我们可以使用android:largeHeap来缓解OutOfMemoryError.

只需要在manifest的application节点加入android:largeHeap=“true”即可:

<application 
    android:icon="@drawable/icon"
    android:label="@string/app_name"
    android:largeHeap="true">

Ubuntu adb devices no permissions 问题解决

发表于 2017-10-17 | 更新于 2017-12-15 | 分类于 Android |
字数统计 | 阅读时长

问题:

ubuntu连接一些手机时无法使用adb连接,使用adb devices查看状态为no permissions.

解决:

  • 拔掉手机,在终端使用lsusb命令
  • 插上手机,再次使用lsusb命令

  • 找到新增的usb设备

    例如:Bus 001 Device 010: ID 0bb4:0c87 High Tech Computer Corp.

  • 创建在/etc/udev/rules.d目录创建51-android.rules文件

    touch /etc/udev/rules.d/51-android.rules

  • 在新建的文件中写入配置.

    SUBSYSTEM==”usb”, ATTRS{idVendor}==”0bb4”,MODE=”0666”
    (可以同时配置多条,分号隔开)

  • 修改文件权限

    sudo chmod a+rx /etc/udev/rules.d/51-android.rules

  • 重启udev,adb

    sudo service udev restart
    重新插拔手机
    adb kill-server
    adb devices
    (如果无效,重启计算机)

以上.

参考:

谷歌官方文档:https://developer.android.com/studio/run/device.html

用脑子想想该不该用boolean类型

发表于 2017-10-13 | 更新于 2017-12-15 | 分类于 技术杂谈 |
字数统计 | 阅读时长

boolean作为java的基本类型,只有两个值,true or false.

boolean类型的使用与人的简单思考很符合,上班迟到或者不迟到,中午下雨或者不下雨,晚上加班或者不加班,回家打游戏或者不打游戏…等等.

然而事情并没有这么简单.任何事几乎都可以有一种情况,不知道!!!

举个代码的例子:

功能:一个视频列表展示,已收藏的视频显示一个红心,未收藏的显示一个灰色心.,视频列表在服务端,收藏信息在本地数据库.

简单想用一个boolean刚好可以标识两个状态.

编码:
从服务端获取列表展示.
展示的几个视频从数据库获取是否收藏,更改图表状态.

上述两个过程都是异步的,也就是在展示的时候并不知道是否收藏.也就是多了一个状态 不知道.

所以即使展示状态只有2个,但是编码过程中真正的状态不止两个.

一旦用boolean限制了状态数,后续重构就会感觉蛋疼了.所以用boolean的时候用脑子好好想想.
