开发不同阶段需要用到的Git命令

发表于 2018-09-10 | 更新于 2018-09-10 | 分类于 Git |
字数统计 | 阅读时长

写在前面

基本概念：
工作区：改动（增删文件和内容）
暂存区：输入命令：git add 改动的文件名，此次改动就放到了‘暂存区’
未跟踪的文件（untracked files）：新建的文件，在git add之前的状态

下面是用到的一些占位符：
<branch-name>：分支名
<message>：描述信息
<index>：序号
<commit-id>：git log显示的神秘字符串
file-path：文件路径

开发阶段

切换分支
git checkout <branch-name>

创建并且切换至分支
git checkout -b <branch-name>

将分支推送到远程仓库
git push origin <branch-name>

将工作区的修改加入暂存区
git add <file-path>

将当前目录的工作区修改加入暂存区
git add .

本地提交代码
git commit -m <message>

存储本地修改
git stash

恢复本地修改
git stash pop
git stash apply <index>

修改最近一次提交日志
git commit --amend

合并最近两次提交
git rebase –i HEAD~2
将其中一个pick修改为squash，修改日志，:wq保存，两个commit会合并成一个。

合并代码

将另一个分支的代码合入当前分支，不推荐使用
git merge <branch-name>

将另一个分支的代码合入当前分支，并且保留commit记录
git merge --no-ff <branch-name>

将另一个分支的代码合入当前分支，并且不保留commit记录
git merge --squash <branch-name>

将某一笔提交合入当前分支
git cherry-pick <commit-id>

删除远程分支
git push origin :<branch-name>

发版阶段

创建tag
git tag -a <tag-name> -m <message> <commit-id>

查看branch1有，而branch2没有的commit
git log <branch-1> ^<branch-2>

查看branch2比branch1多的commit
git log <branch-1>..<branch-2>

对比branch1与branch2的差异
git log <branch-1>...<branch-2>

对比branch1与branch2的差异
git log --left-right <branch-1>...<branch-2>

排查问题

查看分支上的git操作
git reflog

查看commit记录
git log

查看文件每一行的最后修改人/时间/commit id
git blame <file-path>

查看某个commit的修改
git show <commit-id>

对比两个commit的差异：
git diff <commit-id-1> <commit-id-2>
加号减号表示commit2相对于commit1的增减。
